# Portable Log Browser

## PRD — Personal Portable Log Ingester/Browser

### Vision

A self‑contained, cross‑platform desktop app (Avalonia, AOT‑friendly) that ingests heterogeneous log files (text, CSV, JSON/NDJSON, ZIP entries) into a structured, queryable store. The UI offers Seq/OpenObserve‑class exploration: **millions of records**, virtualized grid, side detail, **scrollable timeline with histograms**, **faceted filtering**, **query box with TICK‑style period expressions**, and **instant show/hide of contributors**.

### Users

* **Primary**: Senior developer/SRE troubleshooting complex environments; wants fast local tooling for ad‑hoc sessions.
* **Secondary**: Colleagues who consume exported views or curated sessions.

### Goals & Non‑Goals

* **Goals**
  * Ingest large, messy logs with **auto‑detection** (record boundaries, timestamps, CSV delimiter/header, split timestamps).
  * **Damage tolerance**: skip unparseable segments, **audit skips**.
  * **Timezone policy** for timestamp strings without offset (`Z`), with **per‑file exceptions** and **templated rules**.
  * **Transactional re‑ingest** of only affected files/segments (no full rebuild).
  * **Seq/OpenObserve‑like** browse UX at scale (millions of rows).
  * **TICK‑style** period DSL in queries (user types `timestamp IN '...'`).
* **Non‑Goals**
  * Distributed catalog or multi‑writer concurrency.
  * Server deployment. (Single‑user, in‑process engine.)

### High‑Level Solution

* **Storage**: Use **DuckLake** for ACID table format on top of **Parquet** data files with a **SQL catalog** (DuckDB‑backed), which drastically simplifies metadata management and re‑ingest (DELETE+INSERT inside transactions). [\[duckdb.org\]](https://duckdb.org/docs/stable/sql/functions/date), [\[notepad.onghu.com\]](https://notepad.onghu.com/2025/exploring-duckdb-part2/), [\[github.com\]](https://github.com/duckdb/duckdb/issues/600)
* **Query engine**: In‑process **DuckDB**; time functions (e.g., `date_trunc`, `time_bucket`) for timeline binning. [\[questdb.com\]](https://questdb.com/blog/questdb-9-3-2-release/), [\[py-questdb...thedocs.io\]](https://py-questdb-client.readthedocs.io/en/latest/examples.html)
* **Period DSL**: Adopt **QuestDB’s TICK‑style** `WHERE ts IN 'tick_expr'`, implemented via a client‑side rewriter that compiles to **merged UTC intervals** and emits parameterized predicates / a temp interval table. [\[cynkra.github.io\]](https://cynkra.github.io/dd/reference/date_trunc.html), [\[newreleases.io\]](https://newreleases.io/project/github/questdb/questdb/release/9.3.2)

### User Journeys (MVP)

1. **New Session**
    * Pick directory/files/ZIPs → ingest starts.
    * Auto‑detect per file: record boundary, timestamp extractor (incl. splits), timezone policy (default **Local**, overridable).
    * Progress pane shows **skips** and **confidence**; user can adjust detection/policy and **re‑ingest only those files**.
2. **Explore**
    * Timeline histogram → drag to filter.
    * Virtual grid scrolls smoothly through **M+** rows; side panel shows structured fields & raw text.
    * Facets: Levels, Sources (grouped by template tokens: app/env/host), etc.
    * Query box with **field filters** (`level:Error app:api "timeout"`) + `timestamp IN 'TICK'`.
3. **Audit**
    * Skips panel listing damaged segments with reason, preview, and jump‑to‑context.
4. **Carry‑over**
    * Prior session rules/templates are **suggested**, never forced, with **decaying confidence**.

### Acceptance Criteria (MVP)

* Ingests **text, CSV, NDJSON**, and ZIP entries; multiline records supported.
* Auto‑detect achieves ≥95% correct timestamp extraction on typical formats in a 2k‑record sniff.
* **Skips** are persisted with offsets/lines and reasons.
* **Per‑file** timezone override with “remember for similar files” via filename **templates**.
* **DuckLake** store: transactional delete/insert for re‑ingest; time‑travel snapshot for a run (optional UI later). [\[duckdb.org\]](https://duckdb.org/docs/stable/sql/functions/date)
* Virtual grid: sustained **60fps feel** with pagewise fetch; **keyset pagination** (no OFFSET).
* Timeline: responsive histogram; dynamic bin widths using DuckDB date functions. [\[questdb.com\]](https://questdb.com/blog/questdb-9-3-2-release/), [\[py-questdb...thedocs.io\]](https://py-questdb-client.readthedocs.io/en/latest/examples.html)
* TICK DSL: support literals, `$now/$today` (+/‑ arithmetic), `@timezone`, bracketed date ranges `[a..b]`, duration `;6h30m`. [\[cynkra.github.io\]](https://cynkra.github.io/dd/reference/date_trunc.html)
